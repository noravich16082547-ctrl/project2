# DormCRU — เว็บไซต์หาหอพักสำหรับนักศึกษาปี 1 ราชภัฏเชียงราย

## สิ่งที่อยู่ในโปรเจคนี้
โปรเจคนี้เป็น **เว็บไซต์ต้นแบบ (Prototype)** ที่เขียนด้วย HTML + CSS + JavaScript ล้วน
ไม่มีการใช้ framework ฝั่งเซิร์ฟเวอร์ ครอบคลุมฟีเจอร์ตามที่ระบุไว้ครบทั้ง 3 ฝั่ง:

- `index.html` — หน้าแรก ค้นหา/กรองหอพัก (ราคา, ประเภทห้อง, ประเภทหอ, ระยะทางจากประตูมอ) + QR โค้ด
- `dorm-detail.html` — รายละเอียดหอพัก, แกลเลอรีรูป, สิ่งอำนวยความสะดวก, แผนที่ Google Maps ฝัง, เลือกประเภทห้องเพื่อจอง
- `login.html`, `register.html` — ระบบสมาชิกนักศึกษา
- `wishlist.html` — รายการหอพักที่บันทึกไว้
- `booking.html` — จองห้อง + อัปโหลดสลิปมัดจำ + ดูสถานะการจองของตัวเอง
- `admin-login.html`, `admin-dashboard.html` — ระบบหลังบ้านสำหรับเจ้าของหอพัก/แอดมิน (ภาพรวม, จัดการห้อง/ราคา/ห้องว่าง/รูปภาพ, ตรวจสอบสลิปและยืนยัน/ปฏิเสธการจอง)

บัญชีทดลองฝั่งแอดมิน: **username: `admin`  /  password: `admin1234`**

## ข้อจำกัดสำคัญ (ต้องอ่านก่อนใช้งานจริง)
เพราะโจทย์กำหนดให้ใช้ "HTML" ล้วน โปรเจคนี้จึง **เก็บข้อมูลทั้งหมดไว้ใน localStorage ของเบราว์เซอร์แต่ละเครื่อง**
เพื่อจำลองการทำงานของระบบจอง/สมาชิก/หลังบ้านให้เห็นภาพจริง แต่มีข้อจำกัดคือ:

1. **ข้อมูลไม่ synced ข้ามอุปกรณ์หรือข้ามเบราว์เซอร์** — ถ้านักศึกษาจองจากมือถือ เจ้าของหอที่เปิดจากคอมคนละเครื่องจะไม่เห็นข้อมูลจอง
2. **ไม่มีการเข้ารหัสรหัสผ่านหรือระบบความปลอดภัยจริง** ห้ามใช้กับข้อมูลจริงของผู้ใช้งานจริงจนกว่าจะมี backend
3. **ไฟล์สลิปถูกแปลงเป็น base64 เก็บใน localStorage** ซึ่งมีขนาดจำกัด (~5-10MB ต่อโดเมน) ถ้ามีการจองเยอะมากอาจเต็ม

### ก้าวต่อไปสำหรับใช้งานจริง
เมื่อพร้อมเปิดใช้งานจริงกับนักศึกษาและเจ้าของหอพักหลายคน แนะนำให้ต่อยอดเป็นระบบที่มี backend เช่น:
- ฐานข้อมูลกลาง (MySQL/PostgreSQL/Firebase) แทน localStorage
- ระบบยืนยันตัวตนที่ปลอดภัย (hash รหัสผ่าน, JWT/session)
- อัปโหลดไฟล์สลิปขึ้นที่เก็บไฟล์จริง (เช่น S3, Firebase Storage) แทนการฝัง base64
- แจ้งเตือนผ่าน LINE Notify / อีเมลเมื่อสถานะการจองเปลี่ยน

## วิธีเปิดใช้งาน (ทดสอบในเครื่อง)
เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์ได้ทันที (ดับเบิลคลิก หรือลาก drop ใส่เบราว์เซอร์)
ลองค้นหา/กรองหอพัก สมัครสมาชิก จองห้อง และเข้าระบบหลังบ้านด้วยบัญชีทดลองด้านบนได้ครบทุกฟังก์ชัน

## ขั้นตอนการอัปโหลดขึ้น GitHub และเผยแพร่บน Vercel

1. สร้าง GitHub Repository ใหม่ในบัญชีของคุณ (เช่นตั้งชื่อ `dormcru`)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้นไปยัง Repository นั้น (รวมไฟล์ `vercel.json` ด้วย)
3. สมัครและเข้าสู่ระบบ [Vercel](https://vercel.com)
4. คลิกปุ่ม **"Add New" > "Project"** แล้วเลือกเชื่อมโยงกับ GitHub Repository ที่เพิ่งสร้าง
5. ในขั้นตอนตั้งค่า Project บน Vercel:
   - ตรวจสอบให้มั่นใจว่า **Framework Preset** ถูกระบุเป็น **Other** หรือตรวจพบเป็น HTML/CSS/JS อัตโนมัติ (โปรเจกต์นี้ไม่มีขั้นตอน build ใดๆ)
   - คลิกปุ่ม **Deploy**
6. เมื่อ Deploy สำเร็จ คุณจะได้ลิงก์ URL ของเว็บไซต์ DormCRU ใช้งานได้จริงทันที พร้อมแชร์ให้น้องปี 1 หรือเจ้าของหอพักเข้าใช้งานได้เลย
7. ไฟล์ `vercel.json` ที่แนบมาเปิดใช้งาน Clean URLs ให้อัตโนมัติ (เช่น `/login` แทน `/login.html`) และมี URL สั้นๆ ให้ใช้ เช่น `/admin` (ไปหน้าล็อกอินเจ้าของหอ) และ `/dorm/d001` (ไปหน้ารายละเอียดหอพัก id นั้นๆ)

## แผนที่และรูปภาพ
- แผนที่ใช้ Google Maps Embed (ไม่ต้องใช้ API key) พิกัดตั้งต้นอิงพื้นที่รอบมหาวิทยาลัยราชภัฏเชียงราย ควรแก้พิกัดจริงในไฟล์ `assets/js/data.js`
- รูปภาพตัวอย่างหอพักใช้ placeholder จาก picsum.photos ให้เจ้าของหอพักเข้าหลังบ้านเพื่อเพิ่มลิงก์รูปจริงแทนได้ในเมนู "จัดการห้องพัก"
- QR โค้ดหน้าแรกและหน้าแอดมินสร้างผ่านบริการรูปภาพสาธารณะ (api.qrserver.com) แบบ `<img>` ธรรมดา แทนการโหลดไลบรารี JavaScript จาก CDN ภายนอกแบบเดิม เพื่อไม่ให้เว็บทั้งหน้าล่มหากโหลด CDN นั้นไม่สำเร็จ (ดูหัวข้อ "การแก้ไขจากเวอร์ชันต้นแบบ" ด้านล่าง)

## การแก้ไขจากเวอร์ชันต้นแบบ (Bug Fix Log)
เวอร์ชันต้นแบบมีปัญหา: หน้า `index.html` และ `admin-dashboard.html` โหลดไลบรารี QR code จาก
CDN ภายนอก (`cdnjs.cloudflare.com`) แบบ synchronous แล้วเรียก `new QRCode(...)` ทันที
หากโหลดไลบรารีนี้ไม่สำเร็จ (ถูกบล็อกโดย ad-blocker, ไฟร์วอลล์เครือข่าย, หรือ CDN มีปัญหาชั่วคราว)
จะเกิด `ReferenceError: QRCode is not defined` ซึ่งหยุดการทำงานของโค้ด JavaScript ที่เหลือทั้งหมด
ใน `<script>` บล็อกเดียวกัน — ทำให้รายการหอพักในหน้าแรกและแดชบอร์ดหลังบ้านไม่แสดงผลเลย (หน้าว่างเปล่า)

**วิธีแก้ที่ใช้ในเวอร์ชันนี้:** เปลี่ยนจากไลบรารี JavaScript ภายนอกมาใช้ `<img>` ที่ชี้ไปยัง
บริการสร้างรูป QR code แทน (ซึ่งถ้าโหลดรูปไม่สำเร็จ จะแค่ไม่แสดงรูป ไม่ทำให้ JavaScript พัง)
พร้อมทั้งย้ายโค้ดส่วน QR ไปไว้ท้ายสุดและครอบด้วย `try/catch` เพื่อความปลอดภัยสูงสุด
ทดสอบแล้วว่าเว็บไซต์ทำงานได้ครบทุกฟังก์ชัน (ค้นหา/กรอง/สมัครสมาชิก/จอง/อัปโหลดสลิป/อนุมัติหลังบ้าน)
แม้ในกรณีที่ทรัพยากรภายนอกทั้งหมดโหลดไม่สำเร็จ
